# Fixture Corpus

Archive fixtures live outside platform source trees so bridge, Android, and iOS tests can share the same corpus.

## Layout

```text
fixtures/
  archives/
    required/      launch-scope archive fixtures
    optional/      additional compatibility fixtures
    hostile/       unsafe paths, duplicate paths, damaged archives, zip-bomb-like bounded samples
  metadata/        fixture passwords and expected outcomes for non-secret test data
  platform/        provider, permission, low-storage, and cloud-unavailable simulation notes
```

## Rules

- Do not check in copyrighted, private, or sensitive files.
- Fixture passwords are test data only; keep them in `fixtures/metadata/`.
- Hostile fixtures must be safe to store and run in CI.
- Prefer tiny deterministic archives; large fixtures should be generated by scripts unless the binary corpus is intentionally versioned.
- Do not place real Android `content://` URIs, iOS security-scoped URLs, or provider tokens in fixtures.
